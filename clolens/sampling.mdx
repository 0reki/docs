## 采样策略

CloLens 在服务端对链路进行尾采样，列表中仅显示被保留的链路：

| 保留条件 | 说明                            |
| -------- | ------------------------------- |
| 错误     | 有 Span 出错的链路全量保留      |
| 慢请求   | 总耗时 ≥ 1000 ms 的链路全量保留 |
| 正常     | 按链路 ID 稳定哈希，保留约 10%  |

<Info>
  未命中保留条件的正常链路会被丢弃，不计入列表，请求总量可参考派生的 RED 指标。
</Info>

## RED 指标与告警

CloLens 会持续聚合以下派生指标，可用于配置告警规则：

| 指标                    | 说明         |
| ----------------------- | ------------ |
| `trace.request_rate`    | 请求速率     |
| `trace.error_rate`      | 错误率       |
| `trace.error_count`     | 错误次数     |
| `trace.duration_avg_ms` | 平均响应耗时 |
| `trace.duration_max_ms` | 最大响应耗时 |
| `trace.duration_p95_ms` | P95 响应耗时 |

配置方法见 [告警规则](/alerts/rules) 。
