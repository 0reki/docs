---
title: "概述"
---

告警系统为 Clonitor 持续监控设备指标，并针对指标异常的设备发送通知。

## 核心概念

| 概念   | 说明                                   |
| ------ | -------------------------------------- |
| 规则   | 监控的目标和指标、触发器和接收器       |
| 静默   | 在特定时间窗口内屏蔽告警通知           |
| 抑制   | 在特定告警触发时，自动压制其他相关告警 |
| 接收器 | 告警通知的接收者                       |

## 级别

我们建议在 Clonitor 中使用告警系统时按照如下语义配置告警规则。

| 级别 | 说明         |
| ---- | ------------ |
| 信息 | 仅提示       |
| 警告 | 需要关注     |
| 严重 | 需要立即处理 |

## 状态

| 状态 | 说明                       |
| ---- | -------------------------- |
| 告警 | 条件持续违反，告警激活中   |
| 恢复 | 条件已恢复，告警关闭       |
| 静默 | 被静默规则压制，不发送通知 |
| 抑制 | 被抑制规则压制，不发送通知 |

## 生命周期

<Steps>
  <Step title="告警触发">采集器上报的指标满足规则中定义的触发条件</Step>
  <Step title="持续计时">触发条件持续满足，直到到达规则中定义的时间窗口</Step>
  <Step title="告警激活">告警进入激活状态，向接收器发送告警通知</Step>
  <Step title="指标恢复">采集器上报的指标不再满足触发条件</Step>
  <Step title="告警关闭">告警进入恢复状态，向接收器发送恢复通知</Step>
</Steps>

历史告警始终会保留在系统中，可通过告警历史查看完整记录。
