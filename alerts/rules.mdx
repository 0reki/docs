---
title: "告警规则"
---

告警规则是告警系统的核心，定义告警系统所需的一系列条件。

## 目标

包含设备的组。

## 指标

1. **主机指标**

| 指标               | 说明                      |
| ------------------ | ------------------------- |
| `cpu_usage`        | CPU 使用率（%）           |
| `mem_usage`        | 内存使用率（%）           |
| `disk_usage`       | 磁盘使用率（最高分区，%） |
| `tcp_connections`  | TCP 连接数                |
| `load.load1`       | 1 分钟系统负载            |
| `load.load5`       | 5 分钟系统负载            |
| `load.load15`      | 15 分钟系统负载           |
| `disk_read_bytes`  | 磁盘读取速率（bytes/s）   |
| `disk_write_bytes` | 磁盘写入速率（bytes/s）   |
| `net_rx_bytes`     | 网络接收速率（bytes/s）   |
| `net_tx_bytes`     | 网络发送速率（bytes/s）   |

**链路与服务指标**

| 指标                    | 说明                |
| ----------------------- | ------------------- |
| `trace.request_rate`    | 请求速率（次/分钟） |
| `trace.error_rate`      | 错误率（%）         |
| `trace.error_count`     | 错误次数            |
| `trace.duration_avg_ms` | 平均响应时长（ms）  |
| `trace.duration_max_ms` | 最大响应时长（ms）  |
| `trace.duration_p95_ms` | P95 响应时长（ms）  |

还包括插件的自定义指标。

## 函数

Clonitor 支持使用以下函数对指标进行计算。

| 函数     | 说明                           |
| -------- | ------------------------------ |
| `last`   | 最新一个采样值                 |
| `avg`    | 窗口内的平均值                 |
| `min`    | 窗口内的最小值                 |
| `max`    | 窗口内的最大值                 |
| `rate`   | 最近两次采样的每秒变化量       |
| `nodata` | 在窗口内未收到任何新数据时触发 |

## 比较运算符

可使用大于、小于、大于等于、小于等于、等于和不等于。

## 多条件组合

一条规则可以配置多个条件，条件之间可以使用与和或运算连接。

## 持续时长

一些条件可能短暂超出限制的值，但这往往不构成威胁。因此条件可以配置持续时长，表示容许超出限制值的时间，在这段时间内，超过限制的指标不会被报告。此值的范围在 1 秒到 86400 秒。
