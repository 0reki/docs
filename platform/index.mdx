---
title: "概述"
---

平台能力是 Clonitor 连接设备、接收数据、展示运行状态并触发告警的核心部分。Dashboard 面向用户提供管理入口，Agent API 面向设备接收运行数据。

## 结构

平台能力主要由三类入口组成：

| 模块 | 作用 |
| --- | --- |
| 设备接入 | 创建注册凭据，让 Agent 注册成平台中的设备。 |
| 状态与监控 | 接收心跳、主机指标、服务信息和调用链数据。 |
| 告警管理 | 基于指标和规则评估告警，并通过接收器发送通知。 |

这些入口共同围绕设备展开：设备先注册到平台，再持续上报运行数据，平台根据数据展示状态并触发告警。

## 工作原理

<Steps>
  <Step title="创建设备注册">
    管理员在 Dashboard 中创建注册凭据。平台生成一次性 enrollment token，并返回安装命令。
  </Step>

  <Step title="Agent 注册设备">
    Agent 使用 enrollment token 调用注册接口。平台创建 Host 记录，并返回后续请求使用的 agent token。
  </Step>

  <Step title="持续上报数据">
    Agent 使用 Bearer token 调用心跳、指标、服务和调用链接口。平台更新设备在线状态和运行数据。
  </Step>

  <Step title="展示和告警">
    Dashboard 从数据库读取设备、服务、指标和告警事件。告警评估任务根据规则判断是否创建或恢复告警。
  </Step>
</Steps>

## 设备接入

设备接入负责把一台运行 Agent 的机器变成平台中的 Host。注册凭据有有效期和使用次数限制，明文 token 只在创建响应中返回。

详情见 [设备接入](/platform/devices)。

## 状态与监控

状态与监控围绕 Agent 上报的数据展开。心跳维持设备在线状态，主机指标支撑设备详情页，服务数据和调用链用于观察具体应用的运行情况。

详情见 [状态与监控](/platform/status) 和 [服务监控](/platform/services)。

## 告警管理

告警管理基于平台已接收的指标和规则工作。规则定义目标、指标、阈值和通知方式，评估任务负责创建、恢复和通知告警事件。

详情见 [告警管理](/platform/alerts)。
