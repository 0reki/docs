---
title: "插件分配"
---

插件分配是插件被下发到设备必需的过程。插件分配的基本单元是组。

你可以在创建插件时进行插件分配，也可以在插件创建完成后进行分配。插件分配不是创建插件必需的过程。

## 功能释义

Clonitor 支持多种组合的分配。由于插件分配的基本单元是组，这允许对组进行多样的组合以达成分配目标。每个组的分配模式都可以选择包含或排除，这会为对应的项提供对应的功能。排除的优先级相对包含更高。

### 必需

这些插件会被设备自动安装，设备上的用户无法跳过或拒绝。

### 可用

这些插件可被设备选择是否安装。Clonitor 不会自动推送这些插件。

### 卸载

从目标设备上主动移除该插件。该插件是否被其他策略分配，此分配下的设备都会被执行。

## 插件状态

Clonitor 提供设备状态概述和设备状态详细信息。设备状态概述包含以下几种状态：

| 状态   | 说明                                     |
| ------ | ---------------------------------------- |
| 已分配 | 分配已完成，插件正常运行。               |
| 执行中 | 插件包已下发，正在应用插件。             |
| 失败   | 插件包在应用时出现问题。                 |
| 离线   | 分配的组中包含离线的设备，无法下发插件。 |
| 挂起   | 分配已应用到平台，等待下发到设备。       |
